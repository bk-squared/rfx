## What shipped

### Memory-efficient inverse design (issue #31)

The NU forward path now exposes three composable memory knobs that move
peak AD-memory on a 608 k-cell × n_steps = 10 000 patch job from
**184 GB (OOM on a 24 GB GPU) down to ~0.3 GB**.

| kwarg | behaviour |
| ----- | --------- |
| `checkpoint_every=K` | scan-of-scan with `jax.checkpoint` between K-step segments (primary memory win; see #31 VESSL 369367233490 evidence) |
| `n_warmup=N` | first N steps run under `stop_gradient`; AD tape only for the trailing optimize window (#40) |
| `emit_time_series=False` | skip per-step probe sampling when the objective is NTFF / S-param only (#31) |

Companion: `estimate_ad_memory(n_steps, checkpoint_every=K)` now returns
an `ad_segmented_gb` field matched to VESSL measurements within ~25 %
(#39). The legacy `ad_checkpointed_gb ≈ 4 × forward_bytes` is retained
for back-compat but documented as unreliable on the NU path.

### NU physics correctness (issue #48)

Two interacting bugs caused the NU patch antenna FFRP to peak at θ = 88°
(grazing) instead of broadside:

1. `Box._axis_mask` used a global `dc = coords[1] - coords[0]` (first-cell
   size) as the thin-sheet threshold. On a graded-dz axis the first cell
   is CPML-padded 1 mm, so a 0.25 mm PEC sheet in a fine region got
   snapped onto two or three cells.
2. `smooth_grading` inserted transition cells that shifted the user's
   substrate z-coordinates away from the fine-cell block, violating the
   Meep/OpenEMS "metal on a cell edge with symmetric neighbours" rule.

Fixes:
- `Box._axis_mask` now uses a local cell size at the geometry's midpoint.
- `smooth_grading` gains `preserve_regions=[(lo, hi)]` so callers can
  opt in to the thin-PEC-safe convention.
- New preflight check `_validate_thin_metal_on_nu_mesh` warns when a
  thin PEC lies in a cell with asymmetric neighbours.
- `scripts/issue31_patch_validation.py` reconstructs dz explicitly so
  ground, substrate, patch land on cell edges.

After the fix the NU patch antenna produces θ ≈ 1°, broadside_ratio
0.995, Q ≈ 28, f_res 2.65 GHz (+9.3 % vs Balanis 2.42 GHz). The +9.3 %
is the **known rfx coarse-mesh discretization bias** documented in
`docs/research_notes/2026-04-11_crossval12_patch_antenna.md`: at
dx = 1 mm, rfx reads ~+8 % high, OpenEMS reads ~−8 % low, and both
straddle the Balanis TL model (whose own uncertainty is ~5-8 % for
finite ground planes). Same prior crossval shows both tools converge
to ~1 % at mesh_res ≤ 0.5 mm. No regression vs v1.5; bias-sign pattern
is intact.

### Other features

- `minimize_s11_at_freq(target_freq, port_probe_idx)` — single-frequency
  S11 proxy for the differentiable path (issue #50).
- `rfx.visualize.visualize_structure(sim)` and `visualize_farfield_3d(result)`
  — plotly-based 3D dashboard with legend-toggleable PEC / dielectric /
  source / NTFF / CPML overlays (issue #38).
- `rfx.ntff_sweep.ntff_sweep(sim, freqs, batch_size)` — batched
  many-frequency NTFF accumulation (issue #43).
- Preflight thresholds now physics-based rather than cell-count (issue
  #37): PEC < 3 cells is a thin sheet (silent); dielectric warning is
  cells per λ_eff rather than cells per extent.
