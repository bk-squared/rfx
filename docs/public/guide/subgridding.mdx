## Overview

rfx exposes z-slab SBP-SAT refinement through `Simulation.add_refinement(...)`.
The production API is intentionally conservative: it validates the physical
support envelope before running and rejects cases whose interface equations have
not been derived.

Supported with `validation="production"`:

- guarded one-sided full-x/y z refinement slabs that touch exactly one physical
  z boundary,
- PEC/no-CPML boundary termination, plus a narrow mixed-boundary lane where the
  slab touches a closed PEC z face and only the opposite z face is CPML/UPML,
  with source/probe/port locations at least 0.30 slab spans away from the
  remaining artificial z interface,
- vacuum and static nondispersive-material source/probe diagnostics inside
  that guarded envelope,
- no material, PEC-geometry, or conductivity jump at artificial coarse/fine interfaces,
- single-cell lumped or axis-aligned wire impedance ports inside that guarded
  envelope,
- interior NTFF boxes inside that guarded envelope, with finite public
  `ntff_data`/far-field post-processing support,
- a narrow static PEC-geometry lane where a PEC object may touch the remaining
  artificial z interface inside the guarded one-sided PEC/no-CPML envelope,
- a narrow local x/y window lane only when the boundary is PEC/no-CPML and
  every soft source is exactly centered in the local x/y fine window.

Local x/y windowed refinement (`xy_margin`) is **not** broadly supported.  The
only production lane is the central soft-source envelope: PEC/no-CPML, at least
one soft source, every soft source at x/y local-window fractions `(0.5, 0.5)`,
and source/probe locations satisfying the guarded local-window margin.  The
runner co-injects those sources on the overlapping coarse shadow grid by
default.  Off-centre sources, CPML, ports, or broader local-x/y layouts still
fail closed with `xy_windowed_external_crossval_blocked`.

Centered/two-interface slabs remain research diagnostics.  Static material
subgridding is limited to the guarded one-sided envelope; material jumps at an
artificial interface and dispersive/nonlinear materials are still rejected.  A
narrow guarded PEC/no-CPML public path now accepts static PEC geometry touching
the remaining artificial z interface after direct uniform-reference waveform
gating, but broader PEC-interface geometry remains outside the production
claim.  The
fresh Meep/openEMS external crossval covers the guarded frequency envelope
across both z-boundary sides, refinement ratios `2` and `3`, guarded margins
`0.30`/`0.35`/`0.45`, and two cavity aspect ratios.  Separate production
artifacts now support broad raw source/probe waveforms, static nondispersive
materials, lumped/wire-port `Result.s_params`, guarded interior
NTFF/far-field public-path finite results, and a closed-face/opposite-z-CPML
time-domain path only inside the guarded one-sided envelope.  The local x/y
central soft-source lane has a passing RFX-uniform/openEMS/secondary-Meep
frequency artifact, and a narrow grid-aligned guarded wire-port S11/S21
magnitude artifact now passes against openEMS.  Broader local x/y coverage and
broad calibrated wire-port claims remain blocked; broader wire ports and
material jumps across artificial interfaces remain fail-closed outside the
documented guarded lanes.  Guarded NTFF is now a
finite public path for PEC/no-CPML and the narrow closed-face/opposite-z-CPML
lane, with a normalized-pattern openEMS NF2FF reference artifact for the guarded
case.  This is not broad radiation-power calibration or arbitrary open-boundary
far-field support.  The opposite-z CPML external artifact now passes against an
openEMS mixed PEC/PML reference; a Meep open-boundary trace remains as a
diagnostic reference/observable mismatch, not the promotion gate.  Do not extend
this to
refined faces inside CPML, broad CPML/open-boundary radiation accuracy,
centered/two-interface slabs, off-centre or port-bearing local x/y windows,
material or PEC geometry crossing an artificial interface,
dispersive/nonlinear materials, or general subgridding.

## API

```python
from rfx import Simulation

sim = Simulation(freq_max=10e9, domain=(0.04, 0.04, 0.024), boundary="pec")
sim.add_refinement(z_range=(0.002, 0.024), ratio=2, validation="production")
sim.add_source((0.013, 0.013, 0.012), "ez")
sim.add_probe((0.027, 0.027, 0.014), "ez")

report = sim.validate_subgrid()
report.raise_if_unsupported()
artifact_json = report.to_json()

result = sim.run(num_periods=20)
```

`validation="research"` preserves the older experimental lane for diagnostics
and regression tests, but it is not a production support claim.

`SubgridValidationReport` is intentionally artifact-friendly:

- `report.supported` is the production/research envelope decision,
- `report.support_level` names the exact support envelope,
- `report.errors` and `report.warnings` expose stable issue codes,
- `report.raise_if_unsupported()` fails before JAX compile/run,
- `report.to_dict()` and `report.to_json()` can be stored with validation
  or cross-solver artifacts.

## Support matrix

| Case | `validation="production"` status | Codes |
|---|---:|---|
| Guarded one-sided full-x/y z slab; PEC/no-CPML outer boundary; broad source/probe observables inside the margin envelope | supported | `support_envelope` |
| Guarded one-sided full-x/y z slab touching a PEC z face with only the opposite z face using CPML/UPML; source/probe observables inside the margin envelope | supported as a finite public-path gate, not as generalized open-boundary crossval | `support_envelope` |
| Centered/two-interface full-x/y z slab | rejected in production | `z_slab_requires_guarded_boundary` |
| No `add_refinement(...)` configured | rejected | `no_refinement` |
| Unknown validation mode | rejected | `bad_validation_mode` |
| Ratio is not an integer `>= 2` | rejected | `bad_ratio` |
| `z_range` is unordered, non-finite, or outside the domain | rejected | `bad_z_range` |
| `xy_margin` local x/y window with PEC/no-CPML boundary and every soft source exactly centered in the local x/y window | supported in a narrow central soft-source envelope | `support_envelope` |
| Broader `xy_margin` local x/y windows, including off-centre sources, ports, CPML, or unvalidated source layouts | rejected in production; research-only diagnostic | `xy_windowed_external_crossval_blocked` |
| `xy_margin` local x/y window with source/probes near artificial x/y faces | rejected | `xy_windowed_external_crossval_blocked`, `xy_windowed_margin_too_close` |
| Fine-grid mapping is missing or empty | rejected | `no_region`, `empty_fine_grid` |
| Refined slab face overlaps or touches CPML/UPML | rejected in production, warning in research | `subgrid_overlaps_absorber`, `boundary_terminated_requires_pec_no_cpml` |
| `eps_r`, `sigma`, or `mu_r` jumps across the z-lo artificial interface | rejected | `material_jump_at_zlo_interface` |
| `eps_r`, `sigma`, or `mu_r` jumps across the z-hi artificial interface | rejected | `material_jump_at_zhi_interface` |
| Static material transition inside the validated clearance buffer around the remaining artificial z interface | rejected | `material_transition_near_artificial_interface` |
| Static PEC geometry touching the remaining artificial z interface inside the guarded one-sided PEC/no-CPML voxel-equivalent lane | supported | `support_envelope` |
| Unguarded PEC geometry touching an artificial interface, local x/y PEC-interface cases, or CPML PEC-interface layouts | rejected | `pec_at_zlo_interface`, `pec_at_zhi_interface` or geometry/crossval blocker |
| Single-cell lumped or axis-aligned wire impedance port and `run(compute_s_params=True)` inside the guarded one-sided envelope | supported | `support_envelope` |
| Missing port waveform, mixed source-only S-parameter request, or unguarded impedance-port / S-parameter observable | rejected | `impedance_port_unvalidated` or public request error |
| Source or port outside the refined z slab | rejected | `source_or_port_outside_fine_grid` |
| Probe outside the refined z slab | rejected | `probe_outside_fine_grid` |
| Interior NTFF box and `compute_far_field(...)` inside the guarded one-sided envelope, including the closed-face/opposite-z-CPML finite lane | supported as a finite public-path gate, not as generalized external radiation crossval | `support_envelope` |
| NTFF box outside the fine slab or too close to the artificial interface | rejected | `ntff_unvalidated`, `ntff_box_outside_fine_grid`, `ntff_box_margin_too_close` |
| DFT plane, waveguide port, Floquet port, TFSF, or lumped RLC with subgrid | rejected | `dft_plane_unvalidated`, `waveguide_port_unvalidated`, `floquet_port_unvalidated`, `tfsf_unvalidated`, `rlc_unvalidated` |
| Static nondispersive material inside the guarded one-sided envelope with no artificial-interface jump and no near-interface transition | supported | `support_envelope` |
| Static material outside the guarded envelope | rejected | `material_weighted_sat_missing` |
| Dispersive or nonlinear material in a production subgrid run | rejected | `dispersive_or_nonlinear_material` |
| Internal diagnostic-only subgrid override is enabled | rejected | `diagnostic_subgrid_override_unvalidated` |

If a case is rejected, the reference lane is a uniform mesh or the
[Non-Uniform Mesh](../nonuniform-mesh/) workflow until the relevant SAT closure
has been derived and validated.

## Material example

Run:

```bash
python examples/crossval/13_subgrid_material_validation.py
```

The example demonstrates:

1. a guarded one-sided vacuum z-slab case that is accepted by production validation,
2. a centered homogeneous dielectric material case that is rejected for the
   correct fail-closed reasons (`z_slab_requires_guarded_boundary`,
   `material_weighted_sat_missing`),
3. a uniform-mesh dielectric cavity reference that matches the analytic TM110
   resonance, proving the material setup is meaningful even though that
   centered subgrid case is not promoted.

For material memory reduction outside the guarded one-sided envelope, prefer
[Non-Uniform Mesh](../nonuniform-mesh/) or a uniform fine-grid reference until
the relevant subgrid closure is validated.

- the production broad vacuum gate
  `subgrid_vacuum_boundary_margin_contract.json` records
  `boundary_margin_contract_production_supported`; guarded 0.30/0.35/0.45
  margins pass the broad `bandwidth=0.8` raw waveform gate with best max RMS
  about `0.0878`,
- the production material gate
  `subgrid_boundary_margin_material_gate.json` records
  `boundary_margin_material_production_supported`; homogeneous dielectric and

- `scripts/subgrid_pec_interface_geometry_gate.py` records

- `scripts/subgrid_centered_eh_side_norm_search.py` checks whether separating

- `scripts/subgrid_centered_side_norm_probe_diagnosis.py` maps which centered
  probes and time windows still fail under the best side-norm diagnostic,
- `scripts/subgrid_centered_xy_spatial_residual_map.py` maps a small XY grid at
  the best centered side-norm diagnostic and shows the residual is spatially
  localized rather than z-fraction-only,
- `scripts/subgrid_centered_face_projection_search.py` compares node-adjoint
  and colocated-sample z-face restriction choices for that spatial residual;
  it improves one median case but still leaves localized strict-gate failures,
- `scripts/subgrid_centered_interface_sync_search.py` tests a structural
  exterior-interface sync variant against the active coarse planes; the sync
  variants are worse and introduce kill-threshold failures,
- `scripts/subgrid_centered_best_transfer_time_window.py` maps time windows for
  the current best centered side-norm/node-adjoint diagnostic; early windows
  pass, but late-window probes exceed the kill threshold, localizing the
  centered residual to late cavity/interface dynamics,
- `scripts/subgrid_centered_best_transfer_late_spatial_map.py` maps that
  q4-late residual spatially; the kill-threshold failures localize to the
  lowest sampled interior z plane and lower/mid x-y cross probes, so the next
  centered attempt should target lower-interface late-time energy balance,
- `scripts/subgrid_centered_lower_interface_scale_sweep.py` tests asymmetric
  z-lo interface scaling; larger z-lo scale removes q4-late kill failures and
  reduces q4 max RMS, but no sampled scale clears both q4-late and full-window
  strict gates,
- `scripts/subgrid_centered_lower_interface_eh_scale_sweep.py` splits that
  lower-interface multiplier between E and H updates; E-only strengthening
  preserves full-window pass count but barely improves q4, while H-side
  strengthening improves q4 but worsens full-window fidelity, so the
  centered blocker remains a coupled phase/energy-transfer problem,
- `scripts/subgrid_centered_lower_interface_pair_scale_sweep.py` splits the
  lower-interface correction by tangential Maxwell pair `Ex/Hy` versus
  `Ey/-Hx`; pair-specific strengthening removes q4 kills and improves q4 more
  than E-only scaling, but still trades away full-window fidelity and remains
  below the production gate,
- `scripts/subgrid_centered_e_h_trace_blend_sweep.py` tests whether E-SAT
  should use pre-H-SAT, post-H-SAT, or blended H traces; pre-H-SAT traces remove
  q4 kills without reducing full-window pass count, but q4 residual remains far
  above the strict gate, so time-centering alone is not closure,
- `scripts/subgrid_centered_phase_pair_combo_sweep.py` checks whether the
  pre-H-SAT timing clue composes with tangential-pair scaling; combined cases
  lower q4 residual further, but only by spending full-window fidelity, so the
  production path still needs a derived two-interface operator rather than
  stacked diagnostic knobs,
- `scripts/subgrid_centered_deferred_h_sat_sweep.py` tests a stronger ordering
  change that withholds H-SAT until after E-SAT; all deferred cases go
  non-finite/kill-blocked, so the useful timing clue is limited to the trace
  used by E-SAT, not deferring the H correction itself,
- `scripts/subgrid_centered_e_h_trace_side_blend_sweep.py` localizes that trace
  clue: z-lo-only pre-H-SAT traces reproduce almost all of the q4-kill removal,
  while z-hi-only traces do not help, so the phase issue is lower-interface
  localized,
- `scripts/subgrid_centered_zlo_trace_pair_combo_sweep.py` combines that z-lo
  trace localization with tangential-pair scaling; it follows the same trade-off
  frontier as the global-pre combo and does not clear the full-window gate,
- `scripts/subgrid_centered_zlo_trace_spatial_filter_sweep.py` splits the z-lo
  pre-H-SAT trace delta into face-mean and spatial-residual parts; the benefit
  lives almost entirely in the spatial residual, not the face mean,
- `scripts/subgrid_centered_zlo_trace_component_filter_sweep.py` applies that
  spatial residual to Hx or Hy alone; each component removes q4 kills only
  partially, while both together are needed for the full residual improvement,
- `scripts/subgrid_centered_zlo_trace_vector_mix_sweep.py` tests simple 2x2
  Hx/Hy orientation mixes; swapping or rotating the residual destroys the
  improvement, so the clue is the identity tangential residual itself,
- `scripts/subgrid_centered_zlo_trace_residual_limit_sweep.py` caps that
  identity residual pointwise against the local pre/post tangential H-trace
  energy norm; it preserves q4 kill removal but does not clear the strict
  centered full/q4 gates, so a derived two-interface closure is still needed,
- `scripts/subgrid_centered_zlo_common_trace_projection_sweep.py` tests whether
  the lower-interface SAT should use one projected common trace instead of
  separate coarse/fine common traces; a coarse-face common trace improves q4
  residuals, while an averaged coarse/fine projected trace is intermediate and
  still does not clear the strict centered gates,
- `scripts/subgrid_centered_common_trace_pair_combo_sweep.py` combines that
  coarse common-trace clue with tangential-pair balancing; it confirms the same
  trade-off frontier remains and post-trace pair strengthening can be
  catastrophically unstable,
- `scripts/subgrid_centered_common_trace_side_norm_sweep.py` recalibrates the
  side coefficients inside the new coarse common-trace family; `c≈1.3, f=1.0`
  improves q4 max while preserving the current full-window pass count, but
  strict centered gates remain blocked,
