## v1.6.3

Per-axis boundary correctness + `distributed=True` thread-through for progressive inverse design.

### Fixes

- **Periodic axes no longer allocate CPML** (issue #68): `set_periodic_axes("xy")` combined with `boundary="cpml"` previously allocated CPML on all six faces, so the absorber fought the periodic wrap-around and corrupted the physics (rfx-TAP ex6 absorber: S11 = 0 dB flat across 2–18 GHz, optimisation gradients stuck). `_build_grid` now computes the effective CPML axis set as `"xyz"` minus `_periodic_axes`; preflight P1.9 no longer false-warns on periodic axes. Parametrised test coverage over all 7 axis subsets.

### New features

- **`optimize()` and `progressive_optimize()` accept `distributed=True`** (issue #69): threads through to each inner `sim.forward(distributed=...)` call. Reject rules (NU-only, no TFSF, no waveguide ports, ≥ 2 devices) are inherited from the `forward()` side. Default `distributed=False` preserves v1.6.2 behaviour.

### Test status

12 new tests (`tests/test_periodic_cpml.py`) + 2 new progressive-distributed tests, all green. No regressions on `tests/test_ris.py`, `tests/test_floquet.py`, `tests/test_api.py`, or the full distributed / NU composition suite.

## v1.6.2

`Simulation.forward(distributed=True)` opt-in for non-uniform meshes + correctness fix for the multi-device gradient path. No physics changes on the single-device path vs v1.6.1.

### New features

- **`Simulation.forward(distributed=True)`** (issue #44): opt-in 2-GPU+ sharded forward + grad on non-uniform meshes. Dispatches through `rfx/runners/distributed_nu.py`, composes segmented remat via `checkpoint_every`, and emits a one-time `UserWarning` summarising the evidence. NU-only in v1.6.2 (DP3). Reject rules (TFSF / waveguide ports / uniform mesh / `distributed=True` on single device) are raised up front.
- **`Simulation.__init__` mesh-as-design-variable gradient** (nu_known_limits #2): tracer-valued `dz_profile` now flows end-to-end from the `Simulation` level. Caller supplies a concrete `domain[2]` (profile sum cannot be host-coerced during tracing); all validation sites (`_validate_mesh_quality`, `_check_numerical_dispersion`, `_validate_thin_metal_on_nu_mesh`, CPML-z advisories) skip host-side scans on tracer input. Downstream `coords_from_nonuniform_grid` / `position_to_index` / `make_current_source` stay tracer-safe.

### Fixes

- **Distributed mixed Debye+Lorentz gradient NaN** (issue #44 Phase 4): `_split_lorentz_coeffs` and `shard_lorentz_coeffs_x_slab` padded `lorentz_coeffs.cc` with `0.0`, producing `1 / cc = inf` at ghost cells and leaking NaN through `0 * inf` into real-cell gradients. Now padded with `1 / EPS_0` (vacuum). Forward stays bit-perfect (PEC x-face zeroes the ghost), backward is NaN-free. Phase 3.5 CPU calibration: `grad_nan_count` 166,392 → 0. Regression guarded by `test_distributed_mixed_dispersion_grad_no_corner_nan`.

### Validation

VESSL evidence (DP5 research note `docs/research_notes/2026-04-17_issue44_distributed_nu_composition.md`):

| Hardware | Case | Config | Fwd (single / N-dev) | Bwd | Peak / dev |
| --- | --- | --- | --- | --- | --- |
| RTX 4090 × 1 (Job 0) | 6.66 M cells | ce = 10, n = 100 | 1.998 s / — | 13.64 s | 1.606 GB |
| A6000 × 2 (Job A) | 10 M cells | ce = 25, n = 100 | 4.10 s / 16.45 s | 19.42 s | 27.24 GB |

Test status: 29 kernel + 17 composition + 2 smoke + 10 NU forward-grad + 4 NU gradient + 34 NU API, all green.

### Notes

- M5 spike xfail sentinel (`jax.grad(jit(checkpoint(shard_map(scan))))`) was removed in this release. The upstream JAX composition remains unfixed, but Phase 2F's `custom_vjp` path is the production segmented-remat implementation — the minimal-reproducer xfail no longer documents a correctness gate.
- Job B (4-GPU scale-out) is parked; REMI infra has no 4-GPU node. The 2 × A6000 Job A evidence satisfies the multi-GPU memory-profile acceptance from the issue body.

## v1.6.1

Preflight hardening + differentiable-path unification. No physics changes vs v1.6.0.

### New features

- **Auto-preflight** (issue #66): `forward()`, `optimize()`, and `topology_optimize()` now call `sim.preflight(strict=False)` at entry and emit a single `UserWarning` summarising any issues. Physics violations surface before the AD tape is built instead of 20-60 min into a run. Opt out via `skip_preflight=True`.
- **`optimize()` routes through `forward()`** (issue #64): the differentiable inverse-design entry point now delegates to `sim.forward()` for uniform/NU dispatch, and forwards the `checkpoint_every` / `n_warmup` / `emit_time_series` kwargs. NU meshes (`dz_profile`, `dx_profile`, `dy_profile`) are now usable with `optimize()` without raising.

### Fixes

- **PEC-in-CPML detection** (issue #61): preflight P1.9 no longer skips PEC entries when scanning for geometry inside the CPML region.
- **Taflove dispersion check**: the cell-ratio heuristic is replaced with a physics-based per-axis phase-velocity error at `freq_max` (application-independent — no more false positives on lossy-FR4 patch configs).
- **Probe/source-in-PEC check** (P1.8): warns when a probe, port, or source location lies inside a PEC volume.
- **NU cell aspect-ratio warning**: non-uniform axes with cell ratios above 2.0:1 now warn with a per-axis Courant-asymmetry line, pinpointing which axis of the dz/dx/dy profile is the offender.
- **Decimated harminv** (`scripts/convergence_grid.py`): Matrix-Pencil SVD is O(N³); the F3 case (dx = 0.25 mm, 20.6 M cells, N_tail ≈ 90 k) hit a 57 GB Hankel and timed out on CPU. The script now decimates the time series before harminv whenever N_tail > 30 k, capped by the Nyquist-safe factor `1 / (4 · f_max · dt)`. Empirical: D up to the Nyquist cap preserves `f_res` to 1e-4 % and `Q` exactly.

### Validation

Convergence sweep (2.4 GHz FR4 patch, F1/F2/F3 at 4:1 aspect) completed end-to-end with the decimated harminv, confirming monotonic convergence:

| Case | dx | f_res (GHz) | Error vs Balanis |
| ---- | -- | ----------- | ---------------- |
| F1 | 1.0 mm | 2.649 | 9.31 % |
| F2 | 0.5 mm | 2.563 | 5.76 % |
| F3 | 0.25 mm (20.6 M cells) | 2.475 | **2.13 %** |

## v1.6.0

Memory-efficient inverse design + NU physics correctness.

### New features

- **`checkpoint_every`** kwarg on the NU `forward()` (scan-of-scan remat). AD memory for reverse-mode scales as `O(sqrt(n_steps))` at the cost of extra forward recomputation (issue #31).
- **`n_warmup`** kwarg: first *n_warmup* steps run under `stop_gradient` so the AD tape only covers the trailing optimization window (issue #40).
- **`emit_time_series`** kwarg: skip per-step probe sampling when objectives are frequency-domain (NTFF, S-params) — removes the `(n_steps, n_probes)` tape entry (issue #31).
- **`ntff_sweep(sim, freqs, batch_size)`**: batched multi-frequency NTFF accumulation — memory ∝ 1/N_batches at N_batches × wall-time (issue #43).
- **`minimize_s11_at_freq`**: single-frequency S11 proxy for the differentiable `forward()` path (issue #50).
- **3D visualization API** (`rfx.visualize.visualize_structure`, `visualize_farfield_3d`): plotly-based interactive HTML with legend-toggleable structure / CPML / NTFF overlays (issue #38).
- **`smooth_grading(preserve_regions=...)`**: keep user-specified fine-cell blocks intact while smoothing adjacent transitions (issue #48, Meep/OpenEMS thin-PEC convention).

### Fixes

- **`Box._axis_mask` per-cell `dc`** on non-uniform axes: thin PEC sheets no longer rasterize onto two or three cells inside a graded-dz region (issue #48 — caused FFRP to peak grazing instead of broadside).
- **`estimate_ad_memory`** adds `ad_segmented_gb` and `checkpoint_every` fields that reflect the real memory footprint of the segmented scan — the legacy `ad_checkpointed_gb ≈ 4 × forward_bytes` heuristic is documented as unreliable on the NU path (issue #39).
- **Preflight thresholds** switched from cell-count heuristics to physics-based rules: PEC extent < 3 cells is a thin sheet (silent); dielectric warning is cells per λ_eff rather than cells per extent (issue #37). Also new warning for thin metal on NU with asymmetric neighbouring cells (issue #48).
- **Lumped-port / harminv alignment** improved on the NU patch antenna after the rasterization fix — S11 dip and harminv f_res now agree within 1 % at Q ≈ 30 (issue #46).

### Validation

- NU patch antenna (2.4 GHz FR4, dx = 1 mm, dz_sub = 0.25 mm, lossy FR4) reaches **θ ≈ 1° broadside peak** with **Q ≈ 28**. Harminv f_res **2.65 GHz vs Balanis 2.42 GHz (+9.3 %)** — within the known coarse-mesh +8 % rfx bias documented in [`docs/research_notes/2026-04-11_crossval12_patch_antenna.md`](../../research_notes/2026-04-11_crossval12_patch_antenna.md) (rfx and OpenEMS straddle the Balanis value with ±~8 % opposite-sign discretization biases at dx = 1 mm; both converge to ~1 % at mesh_res ≤ 0.5 mm).
- Full NU test suite (50+ tests: `tests/test_nonuniform_*`, plus new preflight / smooth / n_warmup / visualize / ntff_sweep / s11_at_freq pins) passes on both CPU and RTX 4090 GPU (VESSL).
