Cross-solver checks are useful for confidence, but they should stay scoped to the workflow being checked.

## Current public anchors

| Script | Role |
|---|---|
| `examples/crossval/05_patch_antenna.py` | current practical patch cross-check |
| `examples/crossval/11_waveguide_port_wr90.py` | rectangular waveguide-port workflow |

## How to read cross-checks

- A cross-check supports the specific geometry, mesh, source, and observable it exercises.
- It is not a blanket guarantee for every related RF design.
- If a result comes from an workflow outside the documented support scope, say so.

## Broader checks

The repo also contains tests and scripts for cavity, waveguide, material, and port behavior. Public pages should summarize outcomes at the feature level and avoid run-log detail.
