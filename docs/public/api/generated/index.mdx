This page is the subordinate generated reference layer.

Use the curated pages in [API Reference](../) first. If you need a symbol
inventory or a lower-level object that has not been promoted there yet, this is
the route that should eventually carry it.

## Public rule

- curated public API = claims-bearing and intentionally narrow
- generated API = subordinate and symbol-complete
- support and validation wording still lives in the guide and validation
  layers

## Current status

This landing page is the curated entrypoint to the generated layer.
The subordinate generated symbol pages are published as static deep-reference
assets under the same route family, for example:

- `/rfx/api/generated/rfx.html`
- `/rfx/api/generated/rfx/api.html`

Treat those pages as useful lookup surfaces, not as the primary public API
contract.
